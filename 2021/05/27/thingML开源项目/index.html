thingML开源项目

Posted on 2021-05-27 Views:

本文翻译自thingML开源项目的README。

注意：本文中的某些表述和引用资料可能已经不适用于最新版的ThingML。

ThingML方法由 i）一种建模语言，ii）一套工具和 iii）一种方法论组成。建模语言结合了公认的用于设计和实现分布式响应式系统的软件建模结构：

  • 状态图和组件（与UML一致）通过异步消息传递进行通信
  • 独立于平台的指令性动作语言
  • 针对物联网应用的特定结构。

ThingML语言由一套工具支持，其中包括编辑器、转换（如导出到UML）和一个先进的多平台代码生成框架，支持多种目标编程语言（C、Java、Javascript）。方法论记录了物联网服务开发者和平台专家使用的开发过程和工具。

ThingML是在Apache 2.0许可证 下发布的，由挪威奥斯陆SINTEF网络系统和服务部门的@ffleurey和@brice-morin与一个充满活力的开源社区共同开发。ThingML现在由Tellu拥有，但仍然是开源的。

问题、错误报告和功能请求应提交给GitHub上的问题跟踪器

安装ThingML

本节应包含有关获得最新版本的ThingML和如何开始使用的信息。

版本和发布

目前ThingML的推荐版本是2.X.X版本。有标签的版本可在Github发布页获取：

  • 一个独立的JAR，可以在命令行中使用：在发布页面上寻找最新的*.jar文件
  • 一个eclipse更新站点，用于在eclipse中安装ThingML IDE：在发布页面上寻找最新的*.zip文件

1.X.X版本没有得到维护，不应使用（在2017年第三季度停止维护）。2.X.X版本引入了完全重写的基于XText的解析器和编辑器。一些语法上的变化使得为版本1编写的ThingML程序与版本2不兼容。还有一些在版本1中评估过的构造，在版本2中没有重新实现（例如，组、流等）。

ThingML命令行编译器

ThingML命令行编译器是作为一个独立的JAR发布的。它需要Java 8或更新的版本。最新的版本（*.jar）可以在发布页上找到。

命令行工具包含所有的代码生成器和插件，这些都是这个仓库的一部分。

使用方法：java -jar ThingML2CLI.jar将提供使用信息和选项列表。

ThingML基于Eclipse的IDE

  1. 从Eclipse官网安装 “Eclipse IDE for Java and DSL Developers” 。
  2. 从发布页下载最新的更新站点（*.zip）
  3. 启动Eclipse
  4. 从*.zip文件安装ThingML插件: Help -> Install New Software... -> Add... -> Archive...

使用方法：一旦安装了ThingML插件，*.thingml文件将用ThingML编辑器打开。右键单击*.thingml文件，使用HEADS / ThingML子菜单来编译ThingML文件。生成的代码将被放在eclipse项目根目录的thingml-gen文件夹中。记住，只有包含configuration的ThingML文件才能被编译。

基于Docker的发布

警告：目前Dockerub上的镜像没有自动更新。你应该从Dockerfile中构建镜像以获得最新的ThingML版本。

如果你有docker，你可以在thingmleditor仓库或thingmleditor docker hub使用带有Eclipse和ThingML的构建容器。

ThingML程序示例

本节的目标是给出一个开箱即用的例子清单

注意：多年来，我们收集了很多不同的用ThingML制作的样例和项目。然而，他们中的大多数都没有得到维护和更新，以适应最新版本的ThingML。如果你是初学者，这可能会让你感到困惑。

应该开箱即用的例子：

  • 基本的Arduino例子：https://github.com/ffleurey/ThingMLArduinoDemo。”1.Basics “文件夹包含了一组处理数字IO的简单ThingML/Arduino程序。

  • 多平台Breakout游戏（Arduino、Posix C、Java和Javascript）：https://github.com/ffleurey/ThingML-PongTutorial。这个例子展示了如何用ThingML创建平台独立的组件。

  • Arduino <-> Java通信（串行）：https://github.com/ffleurey/ThingML-PressureLogger。这个程序展示了必须创建2个ThingML程序通过串行端口进行通信。一个程序在Arduino上运行，收集传感器的测量数据。另一个是作为一个Java程序运行，通过USB/串口连接从Arduino收集测量结果并显示曲线。这个例子是相当简约的，但应该很容易为你自己的传感器/需求进行自定义。它展示了ThingML的各种功能，如串行通信插件和为ThingML程序添加Maven依赖的可能性。

  • Raspberry Pi GPIOs:https://github.com/ffleurey/ThingML-RPI-Blink。
    几个非常简单的例子展示了如何使用C语言或NodeJS在Raspberry Pi上闪烁LED。

具有已知不相容性的例子:

注意：一些旧的样例可能很容易修改，但其他的样例可能使用了已经从ThingML中删除的功能。

  • 这个仓库中的org.thingml.samples的例子。这个文件夹包含了许多用不同版本的ThingML制作的样例。探索一下来看看可以用ThingML做的不同事情是很好的，但它并不是开始时获得工作样例的地方。

  • 来自HEADS项目的教程：https://github.com/HEADS-project/training/tree/master/1.ThingML_Basics。这个教程基于不再维护的ThingML v1.0。

编译ThingML生成的代码

ThingML编译器生成C、Java、Javascript或Go的平台特定源代码。本节给出了关于如何编译和执行生成的代码的简短指南。

JAVA / Maven

编译成Java时，ThingML会创建一个完整的Maven项目，可以用mvn clean install构建，用mvn exec:java执行。

前提条件：

  • 确保你有一个合适的JDK（JRE是不够的）。
  • 安装Maven

使用Eclipse:

配置Eclipse，使其使用JDK：Window -> Preferences -> Java -> Installed JREs（确保它指向一个JDK）。

  • 右击pom.xml（在thingml-gen/java/your-configuration中）
  • Run as -> Maven build...
  • 在Goals中输入：clean install exec:java

如果Maven声称找不到 “pom.xml “文件，请使用Workspace...按钮改变Run as -> Maven build...窗口中的基本目录，使其指向thingml-gen/java/your-configuration。

网络浏览器的javascript

没什么特别的。在你的系统浏览器（最好是Chrome或Firefox）中打开生成的index.html文件

不要使用嵌入Eclipse中的默认网络浏览器!

Node.JS的Javascript

ThingML创建了一个标准的Node.js包。

前提条件：

  • NodeJS：安装Node.JS

使用Eclipse:

从这个更新网站上：Node.JS - http://www.nodeclipse.org/updates/enide-2015/，安装Features included in Enide Studio .Features Set和Nodeclipse Node.js .Features Set 1.0.2.201509250223

  • 右击package.json（在thingml-gen/nodejs/your-configuration中）
  • Run as -> npm install
  • 右击main.js
  • Run as -> Node Application

可视化UML图（PlantUML）

UML生成器生成的文件是文本文件，使用PlantUML格式。PlantUML将执行布局，并将图表导出为图像。

前提条件：

  • 安装Graphviz
  • 安装PlantUML (如果使用Eclipse插件，则不需要)

使用Eclipse:

使用这个更新网站在Eclipse中安装PlantUML插件：http://hallvard.github.io/plantuml/

  • Window -> Show View -> Other... -> PlantUML

请确保你已经安装了Graphviz。它是Eclipse插件所必需的。如果你在使用PlantUML时遇到问题，请参照http://plantuml.com/eclipse

Posix C

生成的代码是一个完整的C项目，包括一个Makefile文件。

  • 在包含生成的代码的文件夹中打开终端
  • 用make进行编译
  • 用./your-configuration运行

前提条件：

  • 使用对C语言友好的操作系统(如Linux)
  • 安装gcc和make+你正在使用的库。

注意：虚拟机是一种选择。只要不涉及图形/硬件驱动程序，Windows 10上的Ubuntu也可以正常运行。

Arduino C

  • 在Arduino IDE中打开生成的文件
  • 编译
  • 上传至你的电路板

前提条件：

  • 安装Arduino IDE
  • 安装任何你在ThingML程序中使用的Arduino库

关于如何使用Arduino IDE和Arduino板的更多信息，请参看Arduino文档。

Go

  • 在thingml-gen/go/your-configuration打开终端
  • 直接运行程序：go run *.go
  • 编译可执行文件：go build *.go

前提条件

  • 安装适当版本的Go
  • 安装ThingML使用的Go状态机库 go get github.com/SINTEF-9012/gosm。

关于Go包结构的更多信息，请参看Go文档

Teensy C

Teensy编译器已经有一段时间没有被测试了。试用时要注意一些可能的问题。

前提条件：

  • 安装Teensyduino IDE

或者

  • 安装交叉编译的arm工具链

🔷 常见问题

🔹 ThingML代码可以运行在什么地方？

没有地方 ！或者从微控制器到云的几乎任何地方 ！

ThingML文件本身是一个响应式系统的结构（组件）和行为（状态机）的设计规范。它不能被直接执行。

然而，ThingML文件可以被编译（或转换）为Java/JavaScript/C/Arduino源代码，而这些代码又可以平台上被编译和执行。从ThingML生成的代码已经在大量的平台上成功执行：PC Windows/Linux，Raspberry Pi 1、2和3，Intel Edison，Arduino Uno/Mega/Yun/Mini，ESP8266/ESP32，Trinket，Teensy，可能还有其他平台。

🔹 ThingML是如何被形式化的？

ThingML语言被形式化为一个基于EMF的元模型。文本语法被形式化为XText语法。

🔹 ThingML生成的Java/JS/C/Arduino代码并不完全符合我的需求

编译器不是单一的对象，而是围绕ThingML代码生成框架中定义的一组扩展点以模块化的方式实现。

🔹 为什么我不能生成Python/Lua/Ruby/其他各种语言 ？

好吧，这取决于你是否为默认不支持的任何语言实现一个编译器。你还在等什么呢？

🔹 我怎样才能以编程方式处理ThingML模型？

建议1：确保你对ThingML元模型有良好的理解

建议2：看看帮助函数，它们简化了一些典型的处理。

以这种方式保存的模型将在一个大文件中包含原始文件所涉及的所有导入文件。

由于我们迁移到了XText，这个功能目前可能被破坏了。
