Java集合框架概览

Posted on 2020-06-21 Edited on 2020-07-03 Views:

引言

本文整理了Java集合框架的体系，介绍Java集合框架主要包含哪些类和接口，以及它们的底层数据结构。

Java集合框架概览

Java集合主要包括Collection和Map两种，Collection存储对象的集合，Map存储键值对的映射表。

Collection

List

列表。存储的元素是有序的、可重复的。

  • ArrayList：基于Object[]数组实现，支持随机访问。
  • Vector：与 ArrayList 类似，但是是线程安全的。
  • LinkedList：基于双向链表实现，只能顺序访问，但可以快速地增加、删除元素。此外，LinkeList 还可以用于实现栈、队列、双向队列。

Queue

队列

  • PriorityQueue：基于堆结构实现，可以用来实现优先队列。

Deque

双向队列。也可用来实现队列和栈。

  • ArrayDeque：基于数组实现。

Set

集合。存储的元素是不重复的。

  • HashSet：基于哈希表实现，底层采用 HashMap 存储元素，不支持有序性操作。
  • LinkedHashSet：HashSet的子类，基于 LinkedHashMap 实现，内部使用双向链表维护元素的插入顺序。
  • TreeSet：基于红黑树（自平衡的排序二叉树）实现，支持有序性操作，但查找效率不如 HashSet。HashSet 查找元素的时间复杂度为O(1)，TreeSet 查找元素的时间复杂度为O(logN)。

Map

  • HashMap：JDK1.8之前，由数组+链表组成，数组是主体，链表用于解决哈希冲突的问题；JDK1.8以后，底层改为数组+链表或红黑树，当链表中的元素超过阈值（默认为8）时，将链表转换为红黑树以提高查询效率（注：如果当前数组长度小于64，则会优先选择数组扩容，而不是将链表转换为红黑树）。
  • LinkedHashMap：HashMap 的子类，使用双向链表维护元素的顺序。
  • HashTable：与 HashMap 类似，但是是线程安全的。现在可以使用 ConcurrentHashMap 实现线程安全。ConcurrentHashMap 引入分段锁，相比于 HashTable 的全表锁，细化了锁的粒度，提升了效率。
  • TreeMap：基于红黑树实现。
